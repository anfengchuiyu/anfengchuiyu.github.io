PHPHub for Android

该项目是PHPHub网站的安卓客户端，最近全面开源了（包括服务器端和ios、android端）
项目架构采用 MVP 模式

Nucleus

Nucleus is a simple Android library, which utilizes the Model-View-Presenter pattern to properly connect background tasks with visual parts of an application.
翻译（Nucleus是一个简单的安卓第三方库，可以通过MVP模式正确的把后台任务连接到应用程序的视图部分）

介绍（翻译自该项目的README）

自从MVP模式和Rxjava在程序中已经完全取代了Loader和AsyncTask有一段时间了。

但是在程序中仍然有一些瑕疵，即使我们使用这些先进的技术。
比如：在配置改变使后台任务不能继续进行、在进程重启后不能自动的重启后台任务。

虽然大多软件没有这样的功能，但是它们忽视这个就是一个明显的错误，就像是一个坐着等待点击登录按钮的用户，当他进入地铁会因为网络太慢而切换到另一个应用。就这样应用总会产生大量BUG。

Android文档简单的描述了这种问题Processes and Threads - 4. Background process ，”如果一个Activity正确的实现了它的生命周期方法，并且保存的它当前的状态，那么终止它的进程不会对用户造成明显影响，因为当用户又返回到这个Activity时，会回复所有可见状态”。

然而这是不对的–会有明显的影响因为我们不能回复后台任务。程序会恢复它的可见状态，但是会忘记正在做的事情是什么。所以，如果应用要恢复一个progress bar,但不能恢复自身的后台任务，这样用户通常会看到progress bar一直显示的bug。

Nucleus的主要特性（翻译自该项目的README）

  • 为了以防配置改变的影响，Nucleus会自动重连接所有在后台运行的任务到新的View上。这样程序不会忘记它正在做的事情了。

  • 为了以防进程重启，Nucleus会自动重启后台任务。即使是运行在低内存的设备上或者是长时等待后台任务完成，程序仍然是可靠的。

  • 整个库项目已经构建好了，保持着”The Kiss Principle“的思想。任何人都能容易的读懂理解。

  • 想不不依赖与Dagger，你不需要写一个类注入到presenter中。你只需要一个注解就够了。尽管presenter实例化不需要Dagger,但是它们的依赖仍可以被注入。

  • 在Nucleus中，presenter是个额外的类，不依赖View，这样自然就防止了很多由activity context引起的泄露问题。

Scoop

micro framework for building view based modular Android applications.
用于构建基于模块化的视图应用程序的小框架。

###　What do I get?（scoop包含什么？）

  • 1.Router(路由)：允许你控制Screen和backstack之间的跳转。
  • 2.View Controller(视图控制器)：一个新的构建区，用于替换Activity和Fragment
  • 3.Scoops(分层控制)：通过分层管理应用的依赖和生命周期
  • 4.Transitions（动画过渡）：我们提供一组基本的过渡动画，如左滑右滑。此外，你还可以创建自定义动画过渡。

Android-CleanArchitecture

无论从架构还是代码上看，分层都是三层：视图层(Presentation Layer)、控制层(Domain Layer)、数据流层(Data Layer)。
层级之间通过添加接口层作为分隔实现解耦。
简单来说，优点有以下

  • 层次分明，各层级之间都不管对方如何实现，只关注结果；

  • 在视图层(Presentation Layer)使用MVP架构，使原本臃肿的Activity(或Fragment)变得简单，其处理方法都交给了Presenter。

  • 易于做测试，只要基于每个模块单独做好单元测试就能确保整体的稳定性。

  • 易于快速迭代，基于代码的低耦合，只需在业务逻辑上增加接口，然后在相应的层级分别实现即可，丝毫不影响其他功能。

….等等

目前发现的缺点：

  • 由于视图层(Presentation Layer)使用MVP模式，每个有独立逻辑的Activity(Fragment)都拥有独立的Presenter，当View多起来时候Presenter维护起来就显得略麻烦

  • 上手难度比较大，学习曲线比较陡峭

Android 中的 Flux 架构

Flux架构, 顾名思义表示流, 是以数据流为基础，且是单向的数据流。
Facebook使用Flux架构来构建客户端Web应用. Flux架构并不是为移动端设计的, 但是我们仍然可以在Android开发时采用这个思想.

Android MVP 探究与总结

导读：在Android的开发过程中，Activity承担了大量地工作。如果对整个项目十分了解，并且在开发过程中有意识地抽离出了一些通用层的话，维护起来还稍微好一点，但实际上我们经常会遇到这样一些情况：维护并迭代已有地商业项目（可能前几期并不是由你开发）;UI变动极大（是不是经常Crtl+f到处查找？）…

如果按照MVC的项目结构来说，Activity应该是属于View这一层。而实质上，它既承担了View，更多地却包含一些Controller的东西在里面。这对于开发与维护来说极度不利，耦合度大高了

那么有没有什么解决方案呢？答案当然就是本文所讲述的——MVP了
